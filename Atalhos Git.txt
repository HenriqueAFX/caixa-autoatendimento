git add .
-> O "." significa "adicionar tudo" que foi modificado, criado ou deletado na sua pasta.

git add nome_do_arquivo.txt outro_arquivo.py
-> Se você quiser adicionar arquivos específicos.

git commit -m "Escreva aqui uma mensagem clara e concisa sobre as suas alterações"
-> Depois de adicionar os arquivos, você precisa criar um "commit". Um commit é como um instantâneo do seu projeto em um determinado momento, acompanhado de uma mensagem que descreve as mudanças que você fez.

git push origin main
-> Finalmente, para enviar esses commits (seus "pacotes" de mudanças) para o seu repositório no GitHub, você usará o comando push. A primeira vez que você fizer isso, talvez precise especificar para qual repositório remoto e qual branch (ramo) você quer enviar. Geralmente, o branch principal é chamado de main ou master.


Depois de configurar o repositório remoto (origin) e o branch pela primeira vez, você poderá usar apenas:

git add .
git commit -m "Mensagem da sua nova atualização"
git push



Sempre que você fizer alterações no seu código:

git add . (ou git add <arquivos_específicos>)
git commit -m "Sua mensagem descritiva"
git push
